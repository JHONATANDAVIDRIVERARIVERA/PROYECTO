Fase 2: Entendimiento e Ingeniería de Datos

Introducción

La Fase 2, denominada Entendimiento e Ingeniería de Datos, tiene como propósito analizar, preparar y transformar los datos que serán utilizados para el desarrollo del modelo de aprendizaje automático. En esta etapa se estudia la procedencia, la estructura, la calidad y la relevancia de los datos, con el fin de garantizar que el modelo de inteligencia artificial reciba información limpia, coherente y representativa del problema de negocio planteado en la Fase 1.

El éxito de un proyecto de aprendizaje automático depende en gran medida de la calidad de los datos. Por ello, esta fase busca comprender la naturaleza de los conjuntos de datos, identificar patrones, detectar valores atípicos o inconsistentes, y aplicar las transformaciones necesarias para su posterior procesamiento y entrenamiento.

1. Recolección y comprensión de los datos

La primera etapa del entendimiento de datos consiste en identificar las fuentes de información. En este proyecto, las imágenes provienen de un conjunto de datos de residuos sólidos clasificados por tipo de material (plástico, vidrio, metal, papel, cartón y orgánicos). Cada imagen contiene variaciones en condiciones de luz, ángulos, fondos y tamaños, lo que hace necesario un preprocesamiento adecuado.

Se recopilaron más de 2.000 imágenes distribuidas de forma equitativa entre las categorías. Los datos fueron obtenidos de repositorios abiertos como Kaggle y Google Dataset Search, asegurando que sean públicos y aptos para uso académico. Posteriormente, se realizó una inspección visual y estadística para evaluar la diversidad de las muestras, la resolución promedio, los formatos de archivo y la distribución de clases.

2. Limpieza y depuración de los datos

El proceso de limpieza es esencial para eliminar errores o inconsistencias. Se aplicaron las siguientes acciones:

  • Eliminación de imágenes duplicadas o corruptas.
  • Conversión de formatos no compatibles (por ejemplo, de .png a .jpg).
  • Normalización de dimensiones (todas las imágenes se ajustaron a 224x224 píxeles).
  • Verificación de etiquetas incorrectas o inconsistentes.
  • Revisión de la cantidad de muestras por clase para evitar desbalance de datos.

Este paso permitió garantizar que el modelo no aprenda patrones erróneos o sesgos causados por la calidad desigual de las imágenes.

3. Preprocesamiento de datos

Una vez depurados, los datos fueron sometidos a un conjunto de transformaciones automáticas destinadas a mejorar el rendimiento del modelo. Las principales técnicas utilizadas fueron:

  • Escalamiento y normalización: los valores de píxel fueron reescalados en un rango entre 0 y 1.
  • Aumento de datos (Data Augmentation): se aplicaron rotaciones, recortes, espejado horizontal y variaciones de brillo.
  • División del conjunto de datos: los datos se dividieron en tres subconjuntos: 70% entrenamiento, 20% validación y 10% prueba.

Estas etapas son fundamentales para que el modelo generalice correctamente y no dependa de ejemplos específicos vistos durante el entrenamiento.

4. Ingeniería de características

En la ingeniería de características se busca extraer información relevante que facilite el aprendizaje del modelo. Para las imágenes, esto implica utilizar redes neuronales convolucionales que sean capaces de identificar patrones visuales, como bordes, texturas o colores característicos de cada tipo de residuo.

Se utilizó la arquitectura DenseNet121 como base preentrenada sobre el conjunto ImageNet, la cual permite aprovechar características visuales previamente aprendidas y adaptarlas a la tarea de clasificación de residuos. El modelo fue ajustado (fine-tuning) para reconocer las seis categorías definidas.

5. Evaluación de la calidad de los datos

Antes del entrenamiento, se aplicaron métricas de evaluación de calidad tales como:

  • Completitud: 100% de las imágenes con etiquetas válidas.
  • Uniformidad: todas las imágenes con la misma resolución y formato.
  • Equilibrio de clases: diferencia menor al 10% entre categorías.
  • Ausencia de ruido: sin texto ni marcas interferentes.

6. Conclusión de la fase

La Fase 2 permitió establecer una base de datos sólida y homogénea para el desarrollo del sistema de clasificación automática de residuos. A través del entendimiento, limpieza, transformación y validación de los datos, se garantizó la integridad del conjunto de entrenamiento, lo que se traduce en un modelo más preciso, confiable y útil para la toma de decisiones ambientales.

El resultado de esta fase es un conjunto de datos preparado y estructurado que servirá como insumo principal para la siguiente etapa: Fase 3 – Modelado y Entrenamiento del Sistema.
