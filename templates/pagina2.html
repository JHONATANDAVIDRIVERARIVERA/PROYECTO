Fase 2: Análisis y Diseño del Sistema

Análisis del Sistema

En esta fase se realiza un estudio detallado de los requisitos funcionales y no funcionales del sistema, identificando los flujos de información, los procesos principales y las interacciones entre los distintos actores. El análisis permite establecer una estructura sólida que servirá como base para el diseño e implementación.

Modelado de Procesos

Se emplean diagramas de flujo y casos de uso para representar el comportamiento del sistema. Estos modelos facilitan la comprensión de cómo los usuarios interactúan con el software y cómo se gestionan los datos desde la captura de la imagen hasta la clasificación del residuo.

Diseño de la Arquitectura

La arquitectura propuesta se basa en una estructura cliente-servidor, donde la interfaz web permite al usuario subir imágenes y visualizar los resultados de clasificación, mientras que el servidor procesa las imágenes mediante la red neuronal DenseNet121 entrenada con TensorFlow.

Diseño de Base de Datos

Se propone una base de datos relacional que almacene la información de las imágenes, las predicciones realizadas, las fechas de procesamiento y los usuarios que realizaron las pruebas. Esto garantiza trazabilidad y facilita la evaluación del rendimiento del modelo.

Diseño de Interfaz de Usuario

La interfaz está diseñada para ser intuitiva y responsiva, priorizando la accesibilidad y la visualización clara de resultados. Se emplean colores que refuercen la temática ecológica y un flujo de navegación simple para mejorar la experiencia del usuario.

Conclusión

Esta fase permite consolidar la estructura del sistema, integrando los requisitos identificados con un diseño técnico funcional y escalable. Con ello se asegura que la implementación posterior mantenga coherencia con los objetivos del proyecto.
